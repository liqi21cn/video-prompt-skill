## 为什么需要这个 skill

手工为五个 AI 视频平台分别写提示词是个累活，而且容易翻车：

- **Kling** 要中文提示词、`@元素名` 是中文
- **Seedance**（v3.2）要中文四段式结构，`@图N + 名称` 锚定、时间片、七项防崩坏画质尾
- **HappyHorse** 元素名要拼音、每个角色要 2-4 张多角度图
- **Grok** 要短小的英文提示词、用 `@image1`
- **Veo** 完全不用文字标记——纯按顺序传图片数组

这个 skill 把一行剧本一次性生成五个版本（或任意子集），保证语义一致。秘诀：先把每行变成一份中文导演分镜（`sceneDesc`，用 `镜头N[start-end s]:` 格式、每镜六维），再翻译到各平台。

**核心特性：**

- ✅ 五个平台全覆盖（Veo 4、Grok Imagine、Seedance 2.0、HappyHorse 1.0、Kling 3.0 Omni）
- ✅ 子集化：JSON 字段 或 自然语言（"只要 Kling"、"skip Grok"）
- ✅ 语义节奏分析——告别 `N = round(time / 1.5)` 公式
- ✅ **严格 v3.2 镜头格式**——每行输出前自检
- ✅ **每镜六维必填**——任何维度不能默默省略
- ✅ **50 条微表情库**，覆盖 10 个情绪大类
- ✅ 60+ 镜头库，情绪 → 镜头编码映射
- ✅ Kling 物理关键词（`真实重力`、`重心转移`、`头发惯性`）
- ✅ Veo / Seedance 原生音频字段处理
- ✅ **Seedance 2.0 对齐火山引擎官方规范**（中文四段式 + 防崩坏尾）
- ✅ 多镜编排（Kling `multi_prompt`、Seedance 时间片）
- ✅ 流式 JSONL 输出，方便下游解析

## v3.2 更新点

相对 v3.1 的四个硬约束升级：

### 1. 镜头格式强约束

`sceneDesc` 里每个镜头必须用 `镜头N[start-end s]:` 格式——阿拉伯数字、半角方括号、半角 `s`、半角冒号+空格。旧的 `△[time]` 形式**直接判为错误**。

```diff
- △[0-2s] 中景，赵长安 缓慢盘膝坐下…
+ 镜头1[0-2s]: 中景，赵长安@图片2 缓慢盘膝坐下，眉心微皱呼吸放缓…
```

### 2. 每镜六维必填

每个镜头描述必须明确包含六维：

| # | 维度 | 必备要素 |
|---|---|---|
| ① | 景别 | 远景 / 全景 / 中景 / 近景 / 特写 / 极近特写 |
| ② | 主体动作 | 谁 + 做什么 + 怎么做（过程化）|
| ③ | 微表情 | 眼神 / 眉眼 / 嘴角 / 呼吸 / 小动作 |
| ④ | 光影 | 方向 + 色温 + 强度 + 阴影 |
| ⑤ | 运镜 | 推/拉/摇/移/跟/转 + 速度 |
| ⑥ | 音效 | 环境音 / 动作音 / 情绪音 |

如果某个维度确实不适用（比如静止画面没有运镜），明确写 `静止 / 无运镜 / 无动作`——绝不默默省略。

### 3. 50 条微表情库

在镜头描述的 ③ 维度直接引用即可，不必再自己编。

### 4. Seedance 2.0 对齐火山官方规范

旧版：英文 `[Image1]` + `Scene N / lens switch`。**新版**：中文**四段式**结构，对齐火山引擎官方 `byted-ark-seedance-pe v1.0` 规范。

```
[① 全局基础设定]   @图1 角色1名称+描述，@图2 角色2，@图3 场景。
[② 时间片分镜]     0-X 秒：@图N 角色 在 @图M 场景 做什么，<6 维>；
                  X-Y 秒：…；
[③ 音频/台词段]    台词：「…」；音效：…；环境音：…。
[④ 防崩坏画质尾]   4K 高清，细节丰富，<风格>；面部稳定不变形、五官清晰、
                  人体结构正常、动作自然流畅、不僵硬、画面无卡顿、无闪烁。
```

硬规则：
- **`@图N` 必须跟名称**——`@图1 赵长安`，不允许裸 `@图1`（防止中文分词歧义）
- **每个时间片只能有一种运镜**——如果同时有 推入 + 拉远，必须拆成两片
- **画质尾必须包含七项防崩坏**
- `references` 形状改为 `{tag, name, file}`（旧版是 `{type, file, index}`）

| 平台 | 出品方 | 语言 | 引用语法 | 单实体图数 | 多镜 |

## 输入格式

顶层结构：

```json
{
  "platforms": ["kling", "seedance"],     // 可选，默认五个都生成
  "tableData":  [ /* 剧本行 */ ],          // 必填
  "assetLists": { /* 参考素材 */ }         // 必填
}
```

### `tableData[]` — 剧本行

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `time` | number | 是 | 时长（秒）。典型 2-15。|
| `attribute` | string | 是 | `人物·<名>` / `旁白` / `系统` / `环境` / `字幕` |
| `originalText` | string | 是 | 台词 / 旁白 / 系统文字 |

### `assetLists` — 参考素材

```json
{
  "assetLists": {
    "scenes":     [{ "name", "description", "multi_angle_urls": [...] }],
    "characters": [{ "name", "description", "multi_angle_urls": [...] }],
    "items":      [{ "name", "description", "multi_angle_urls": [...] }]
  }
}
```

**`multi_angle_urls` 给几张图：**

| 目标平台 | 推荐图数 |
|---|---|
| 只 Veo / Grok | 1 张正面 |
| 只 Seedance | 1-3 张（每个角度变成一个 `@图N`）|
| 包含 HappyHorse | **2-4 张多角度** |
| 包含 Kling | **4 张多角度**（正 / 侧 / 3/4 / 背）|

完整 schema 与校验规则见 [`references/input-schema.md`](references/input-schema.md)。

## 输出格式

流式 **JSONL** —— 每行一个 JSON 对象。

```json
{
  "time": 4,
  "attribute": "人物·李雷",
  "originalText": "他长叹一声，转身离去。",
  "sceneDesc": "镜头1[0-2s]: 近景，李雷@图片2 ... 镜头2[2-4s]: 中景背影 ...",
  "imageRefs": ["李雷"],

  "veoPrompt":        { "prompt": "...", "reference_images": [...], "negative_prompt": "..." },
  "grokPrompt":       { "prompt": "...", "image_urls": [...], "duration": 4 },
  "seedancePrompt":   { "prompt": "<中文四段式>", "references": [{"tag": "@图1", "name": "...", "file": "..."}] },
  "happyhorsePrompt": { "prompt": "...", "elements": [...] },
  "klingPrompt":      { "prompt": "...", "elements": [...], "multi_prompt": [...] },

  "prompt": ""
}
```

**规则：**

- `time` / `attribute` / `originalText` / `sceneDesc` / `imageRefs` / `prompt: ""` 始终存在
- 平台字段**仅在被请求时**出现——未选中的平台**完全省略**（不写 `null`、不写 `{"skip": true}`）
- `sceneDesc` 使用 **v3.2 严格格式** `镜头N[start-end s]:`（不再用 `△[time]`）
- 一行一个对象——没有数组方括号、没有 markdown 代码块、没有注释

## 平台选择

默认五个平台都生成。可以通过两种方式收窄：

### 方法 1 — 自然语言

| 你说 | 解析为 |
|---|---|
| `只生成 Kling 的提示词` | `["kling"]` |
| `只要 Veo 和 Seedance` | `["veo", "seedance"]` |
| `just Veo` | `["veo"]` |
| `skip Grok and Veo` | `["seedance", "happyhorse", "kling"]` |
| `exclude HappyHorse` | `["veo", "grok", "seedance", "kling"]` |

### 方法 2 — JSON 字段

```json
{ "platforms": ["kling", "seedance"], "tableData": [...], "assetLists": {...} }
```

别名（大小写不敏感）：

| 标准 ID | 识别的别名 |
|---|---|
| `veo` | Veo, Veo 4, Google Veo |
| `grok` | Grok, Grok Imagine, xAI Grok |
| `seedance` | Seedance, Seedance 2.0, 即梦 |
| `happyhorse` | HappyHorse, Happy Horse, 快马 |
| `kling` | Kling, Kling 3.0 Omni, 可灵 |

**两者同时出现时**，JSON 字段优先。

## 严格镜头格式（v3.2）

`sceneDesc` 每个镜头**必须**遵循：

```
镜头N[起始s-结束s]: <六维描述>
```

### 正确 ✓

```
镜头1[0-1.5s]: 中景，赵长安@图片2 缓慢盘膝坐下，眉心微皱呼吸放缓，烛火暖光从右侧打亮面部，缓慢推入聚焦面部，环境音衣料摩擦声
```

### 错误 ✗

| 错误形式 | 错在哪 |
|---|---|
| `△[0-2s] xxxxx` | 缺 `镜头N` + 冒号 |
| `0-2秒：xxxxx` | 缺方括号 + 缺 `镜头N` + 用了中文 秒 |
| `(0-1.5s) xxxxx` | 圆括号而非方括号 |
| `第1段 0-1.5s xxxxx` | 用了 `第N段` 而非 `镜头N` |
| `镜头1：0-1.5s xxxxx` | 时间不在方括号里 |
| `镜头1[0,1.5s]: xxxxx` | 用逗号代替连字符 |
| `Shot1[0-1.5s]: xxxxx` | 用了英文 `Shot` |
| `镜头一[0-1.5s]: xxxxx` | 用了中文数字 |

### 输出前自检（每行执行）

```
□ 每个镜头都以 镜头N[start-end s]: 开头？
□ 每个镜头六个维度都明确出现？
□ 镜头编号从 1 连续？
□ Σ 镜头时长 = `time`？
□ 每个镜头 ≥ 0.8s 且 ≤ 5s？
□ 镜头之间时间连续（无空隙、无重叠）？
□ 微表情是物理化的（不是抽象情绪词）？
```

## 六个必填维度

每个镜头描述必须明确包含六维。顺序可以变，但不能缺。

```
镜头N[start-end s]:
  ① 景别        — 中景
  ② 主体动作     — 赵长安@图片2 缓慢盘膝坐下
  ③ 微表情      — 眉心微皱呼吸放缓
  ④ 光影        — 烛火暖光从右侧打亮面部
  ⑤ 运镜        — 缓慢推入聚焦面部
  ⑥ 音效        — 环境音衣料摩擦声
```

**稀疏维度处理：** 静止画面没有运镜，明确写 `静止 / 无运镜`；纯环境镜头没有主体动作，写 `无主体动作`。绝不默默省略。

## 微表情库

50 条场景标签预设见 [`references/micro-expressions.md`](references/micro-expressions.md)。每条遵循：

> **眼神落点 + 眉眼变化 + 嘴角变化 + 呼吸停顿 + 小动作 + 适用场景**

### 使用方式

1. **情绪 → 分类**（如 委屈 → 9.2.1）
2. **场景 → 条目**（如 "被误会" → #01 强忍委屈）
3. **物理描述照搬到 ③ 维度**，不要带 `#NN` 或场景标签
4. **短镜头可裁剪** —— 眼神落点 + 一个小动作往往足够
5. **复杂情绪可叠加**（如 #18 悔意涌上 + #01 强忍委屈）

### 样例条目

| # | 标签 | 描述 | 适用场景 |
|---|---|---|---|
| 01 | 强忍委屈 | 眼尾泛红但泪不落，眉心轻拧又松开，嘴角向下压住，下巴轻颤，视线从对方脸上滑开像把解释咽回去 | 被误会、背锅、临别强撑 |
| 19 | 破防瞬间 | 原本平静的眼神忽然碎掉，瞳孔轻颤，嘴角努力维持却失败，鼻翼发酸，眼泪在眶里打转但还没掉 | 听到最痛的一句话 |
| 46 | 看到旧物 | 目光落在物件上不动，眉眼迅速柔软又泛酸，呼吸一滞，指尖轻颤想伸出又收回 | 看到亡者遗物、旧信物 |

英文平台（Veo / Grok / HappyHorse）翻译物理观察即可（翻译自然），场景标签名（如 "邪魅试探"）不翻译。

## 节奏分析

镜头数**不**由时长决定。四个维度共同判断：

| 维度 | 看什么 | 效果 |
|---|---|---|
| **动作密度** | 爆发性动词（炸/劈/扑/碎）vs 静态描述 | 极高 → 0.8-1.5s 密集切；低 → 3-5s 长镜 |
| **情绪曲线** | 平静 / 紧张 / 高潮 / 转折 | 映射节奏曲线（慢 / 推进 / 快 / 定格）|
| **对白存在** | 有台词？ | 每句台词留 ≥ 2s 表演时间 |
| **信息密度** | 句子里几个独立视觉点 | 每个主要点至少一个镜头 |

### 硬约束（始终遵守）

- 每个镜头 **≥ 0.8s**（再短肉眼看不到）
- 每个镜头 **≤ 5s**（特殊长镜可 8s）
- ∑ 镜头时长 **= `time` 精确相等**
- 镜头之间无空隙
- 严格 v3.2 格式 + 六维必填

### 例子：同时长，不同切法

```
6 秒，"我想……我们或许该回去了。"
  → 1-2 个长镜：镜头1[0-3s] 说话镜  镜头2[3-6s] 反应镜

6 秒，"他猛地侧身闪开，反手一刀劈下，敌人胸口炸开血花。"
  → 5-6 个密镜：闪身 | 反握 | 挥刀 | 接触 | 血雾 | 余韵
```

完整决策规则见 [`references/rhythm-rules.md`](references/rhythm-rules.md)。

## 镜头库

60+ 镜头分 9 大类，每个有编码：

| 大类 | 主题 | 示例 |
|---|---|---|
| **A** | 推/拉运镜 | `A01` 缓慢推入, `A03` 缓慢拉出, `A08` 希区柯克变焦 |
| **B** | 轨迹运动 | `B04` 弧形绕摄, `B05` 环绕运镜 |
| **C** | 速度/特殊 | `C01` 手持灵动, `C06` 急停定帧 |
| **D** | 角度/构图 | `D01` 荷兰角, `D05` 无力仰角 |
| **E** | 特写/情绪 | `E01` 眼神特写, `E09` 眼泪滑落 |
| **F** | 心理/意识 | `F01` 旋转晕眩, `F02` 模糊转清 |
| **G** | 动作/打斗 | `G07` 子弹时间, `G08` 破门而入 |
| **H** | 转场 | `H04` 回忆叠化, `H06` 长镜压抑 |
| **I** | 特效/风格 | `I06` 水墨晕染, `I11` 魔法释放 |

**选择算法：** 情绪 → 动作 → 风格 级联，冲突时情绪优先。

完整镜头库 + 情绪映射表见 [`references/shot-library.md`](references/shot-library.md)。

## 各平台细节

### Veo 4

- 不要写精确时序（"走 3 秒然后转身"会失败）
- 不要在一条提示词里塞多个故事弧
- **必须带 `Audio:` 段**——否则浪费原生音频能力
- 3 张参考图的上限是硬限制

### Grok Imagine

- **10 秒硬上限**（某些实现 15 秒）
- 提示词**控制在 100 字符以内**效果最好
- 跳过"4K"、"high quality"这类通用形容词——是噪音
- 想要 > 10 秒的连续片段，用帧链（前一片段的最后一帧作为下一片段的输入）

### Seedance 2.0（v3.2 大改）

- **强制中文**（按火山引擎官方规范）——唯一例外是原台词本身是英文时
- **必须四段式结构**：全局 → 时间片 → 音频 → 防崩坏尾
- **`@图N` 必须跟名称**：`@图1 赵长安`，不允许裸 `@图1`
- **每个时间片只能有一种运镜**——推入+拉远要拆成两片
- **画质尾必须**包含七项防崩坏：
  > `面部稳定不变形、五官清晰、人体结构正常、动作自然流畅、不僵硬、画面无卡顿、无闪烁`
- `references` 形状：`{"tag": "@图1", "name": "赵长安", "file": "url"}`
- 最多 9 张图 + 3 个视频 + 3 个音频
- 长镜头会有背景漂移，是已知限制
- **已废弃**形式（用了不会生效）：`[Image1]`、`Scene N`、`lens switch`、`[asset-xxx]`

### HappyHorse 1.0

- **元素名必须用拼音或英文**——中文在 `@` 引用里可能失败
- 多角度（每个元素 2-4 张图）是它的杀手锏
- description 字段对身份识别影响很大，要认真写
- 多镜上限 5 个

### Kling 3.0 Omni

- **用中文写提示词**效果最好
- 物理关键词显著提升真实感：
  - `真实重力` — 重力符合直觉的下落
  - `重心转移` — 行走时的真实重心切换
  - `脚跟先落地` — 自然的脚跟先着地
  - `布料下垂` — 自然的衣料垂感
  - `头发惯性` — 转身时的头发惯性
  - `液体粘度` — 真实的液体粘度
- 触发对白唇形：`@角色名 直视镜头，缓缓开口说道："..."`
- 4+ 镜头时，要同时生成 `multi_prompt` 数组（最多 6 镜）

完整各平台翻译指南见 [`references/platform-adaptations.md`](references/platform-adaptations.md)。

## 常见坑

1. **退回 v3.1 `△[time]` 格式** —— v3.2 严格格式不可妥协
2. **默默省略某个维度** —— 六维必填，不适用要明确写 `静止/无运镜/无动作`
3. **§9.2 库里有却自己编微表情** —— 先翻库
4. **Veo / Seedance 忘了音频** —— Veo 写 `Audio:` 段，Seedance 写环境音/台词段
5. **Seedance 写英文** —— v3.2 按官方规范强制中文
6. **Seedance 用 `[Image1]` / `Scene N / lens switch`** —— v3.2 已废弃
7. **Seedance `@图N` 后没跟名称** —— 必须 `@图1 赵长安`，不能裸 `@图1`
8. **Seedance 单个时间片多个运镜** —— 必须拆成多个片
9. **Seedance 缺 4K + 防崩坏尾** —— 七项防崩坏全部必备
10. **HappyHorse 元素名用了中文** —— 坚持拼音或英文
11. **Kling 多镜动作忘了 `multi_prompt`**（4+ 镜时必须）
12. **时长加起来对不上 `time`** —— 总要核对一遍
13. **场景 `@图片1` 在多个 △ 重复出现** —— 场景只在该行第一个镜头出现
14. **`imageRefs` 里塞场景名** —— 只有人物和道具
15. **写抽象情绪词** —— 永远不要写 `"悲伤"` 或 `"happy"`；要物理化到肌肉/姿态
16. **没选中的平台写 `null`** —— 整个字段省略
17. **别名解析错** —— `可灵` → `kling`、`即梦` → `seedance`、`快马` → `happyhorse`

## FAQ

**Q：我输入的是纯文本剧本，不是结构化 JSON，还能用吗？**
A：可以——skill 会先帮你把文本格式化成 `tableData` 结构再处理。

**Q：角色不在 `assetLists` 里怎么办？**
A：只用名字描述，没有 `@` 引用。所有平台都退回到纯文字描述。skill 会用 `"_no_ref": ["X"]` 标记给你看。

**Q：`time` 是怎么严格执行的？**
A：skill 会验证 ∑(镜头时长) === `time` 精确相等。每个镜头 ≥ 0.8s 且 ≤ 5s。如果时长容不下节奏，会警告。

**Q：不用 Claude 能用吗？**
A：这个 skill 是为 Claude 设计的（通过 `SKILL.md` 描述自动触发）。参考文档也能当人工提示词工程指南用，但自动触发流程需要 Claude。

**Q：skill 会拉取参考图 URL 吗？**
A：不会。URL 可达性是下游的事——skill 只透传 URL。

**Q：怎么更新到新版本？**
A：`cd ~/.claude/skills/video-prompt-skill && git pull`。

**Q：我能加自己的平台吗？**
A：Fork 这个仓库，在 `references/platform-adaptations.md` 加一段，在 `SKILL.md` 的平台 ID 列表里加一条，更新输出 schema，欢迎 PR。

**Q：我有 v3.1 流水线，期望 `△[time]` 格式和 `[Image1]` Seedance，怎么迁移？**
A：v3.2 是 breaking 格式变更。下游解析器需要更新：`sceneDesc` 期望 `镜头N[start-end s]:`，`seedancePrompt.prompt` 期望中文四段式结构。输出 schema **字段名不变**，只是 `sceneDesc` 和 `seedancePrompt.prompt` 的**内容**变了。
